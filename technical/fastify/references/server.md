# Server API Reference

## Factory Options

Create a Fastify instance by calling the factory function with an options object.

```js
const fastify = Fastify({
  logger: true,
  bodyLimit: 1048576,
  trustProxy: true,
  ajv: {
    customOptions: { removeAdditional: 'all' },
    plugins: [require('ajv-formats')]
  }
})
```

### HTTP/HTTPS/HTTP2

```js
// Custom http server
const fastify = Fastify({
  http: {
    requestTimeout: 60000
  }
})

// HTTPS
const fastify = Fastify({
  https: {
    key: fs.readFileSync('key.pem'),
    cert: fs.readFileSync('cert.pem')
  }
})

// HTTP/2
const fastify = Fastify({
  http2: true
})

// HTTP/2 with HTTPS
const fastify = Fastify({
  http2: true,
  https: {
    key: fs.readFileSync('key.pem'),
    cert: fs.readFileSync('cert.pem'),
    allowHTTP1: true
  }
})
```

### Timeouts and Connections

| Option | Default | Description |
|---|---|---|
| `connectionTimeout` | `0` | Milliseconds to wait for the socket to be connected. Sets `server.headersTimeout`. |
| `keepAliveTimeout` | `72000` (72s) | Milliseconds of inactivity before the socket is destroyed. Sets `server.keepAliveTimeout`. |
| `forceCloseConnections` | `'idle'` | Close connections when the server closes. `true` kills all sockets, `'idle'` closes only idle sockets. |
| `maxRequestsPerSocket` | `0` | Max requests per socket (0 = unlimited). Sets `server.maxRequestsPerSocket`. |
| `requestTimeout` | `0` | Milliseconds to receive the entire request from the client. Sets `server.requestTimeout`. |
| `handlerTimeout` | `0` | Milliseconds for a handler to complete before the connection is destroyed. 0 = disabled. |

### Body Parsing

| Option | Default | Description |
|---|---|---|
| `bodyLimit` | `1048576` (1 MiB) | Maximum payload size in bytes. Can also be set per-route. |
| `onProtoPoisoning` | `'error'` | Action when JSON body contains `__proto__`. Values: `'error'`, `'remove'`, `'ignore'`. |
| `onConstructorPoisoning` | `'error'` | Action when JSON body contains `constructor`. Values: `'error'`, `'remove'`, `'ignore'`. |

### Logging

```js
// Enable default Pino logger
const fastify = Fastify({ logger: true })

// Pino logger with options
const fastify = Fastify({
  logger: {
    level: 'info',
    transport: {
      target: 'pino-pretty'
    },
    serializers: {
      req(request) {
        return { method: request.method, url: request.url }
      }
    }
  }
})

// Bring your own logger instance (must be Pino-compatible)
const logger = require('pino')({ level: 'info' })
const fastify = Fastify({ loggerInstance: logger })

// Disable automatic request/response logging
const fastify = Fastify({
  logger: true,
  disableRequestLogging: true
})
```

When `disableRequestLogging` is `true`, the `onRequest` and `onResponse` hooks will not log incoming requests and completed responses. You can add custom logging in hooks instead.

### Server Factory

Use `serverFactory` to pass a custom server instance. When using a custom server factory, `listen` does not manage listening — you must do it in the factory.

```js
const serverFactory = (handler, opts) => {
  const server = http.createServer((req, res) => {
    handler(req, res)
  })
  return server
}

const fastify = Fastify({ serverFactory })

// When using serverFactory, listen on port yourself
fastify.server.listen(3000)
```

### Request ID

```js
const fastify = Fastify({
  // Header to read request ID from (default: 'request-id')
  requestIdHeader: 'x-request-id',

  // Label for request ID in logs (default: 'reqId')
  requestIdLogLabel: 'requestId',

  // Custom request ID generator
  genReqId: (req) => {
    return crypto.randomUUID()
  }
})
```

When `requestIdHeader` is set to `false`, no header is consulted and the ID is always generated by `genReqId`.

### Trust Proxy

```js
// Boolean
const fastify = Fastify({ trustProxy: true })

// Number — trust the Nth hop from front-facing proxy
const fastify = Fastify({ trustProxy: 2 })

// String — trust specific IPs or subnets
const fastify = Fastify({ trustProxy: '127.0.0.1,192.168.0.0/16' })

// Array
const fastify = Fastify({ trustProxy: ['127.0.0.1', '10.0.0.0/8'] })

// Function
const fastify = Fastify({
  trustProxy: (address) => address === '127.0.0.1'
})
```

When enabled, `request.ip` and `request.hostname` reflect the `X-Forwarded-For` / `X-Forwarded-Host` header values.

### Plugin Timeout

```js
// Default is 10000 (10 seconds). Set to 0 to disable.
const fastify = Fastify({ pluginTimeout: 20000 })
```

Milliseconds to wait for a plugin to load before throwing `ERR_AVVIO_PLUGIN_TIMEOUT`.

### Query String Parser

```js
const qs = require('qs')

const fastify = Fastify({
  querystringParser: (str) => qs.parse(str)

```

### Expose HEAD Routes

```js
// Default is true — automatically adds HEAD routes for every GET route
const fastify = Fastify({ exposeHeadRoutes: false })
```

### Constraints

Custom constraints for route matching (e.g., host-based or version-based routing).

```js
const fastify = Fastify({
  constraints: {
    version: {
      storage: () => { /* custom storage */ },
      deriveConstraint: (req) => req.headers['accept-version'],
      validate: (value) => { /* validate constraint */ }
    }
  }
})
```

### Return 503 on Closing

```js
// Default is true — sends 503 when server is closing
const fastify = Fastify({ return503OnClosing: false })
```

### AJV Configuration

```js
const fastify = Fastify({
  ajv: {
    customOptions: {
      removeAdditional: 'all',    // strip unknown properties
      useDefaults: true,          // apply default values from schema
      coerceTypes: true,          // coerce data types
      allErrors: true             // report all errors (not just first)
    },
    plugins: [
      require('ajv-formats'),
      [require('ajv-keywords'), ['transform']]  // [plugin, options]
    ]
  }
})
```

### URL Matching Options

| Option | Default | Description |
|---|---|---|
| `caseSensitive` | `true` | When `false`, `/Foo` and `/foo` match the same route. |
| `allowUnsafeRegex` | `false` | Allow potentially unsafe regex in route parameters. |
| `ignoreTrailingSlash` | `false` | When `true`, `/foo` and `/foo/` match the same route. |
| `ignoreDuplicateSlashes` | `false` | When `true`, `/foo//bar` is treated as `/foo/bar`. |
| `maxParamLength` | `100` | Max characters for route parameter values. Requests exceeding this get a 404. |

### Schema Controller

Override how Fastify builds the schema compiler.

```js
const fastify = Fastify({
  schemaController: {
    bucket: (parentSchemas) => {
      return {
        add(schema) { /* add schema */ },
        getSchema(id) { /* get schema by $id */ },
        getSchemas() { /* get all schemas */ }
      }
    },
    compilersFactory: {
      buildValidator: (externalSchemas, ajvServerOption) => {
        // return a compiler function
      },
      buildSerializer: (externalSchemas, serializerOptsServerOption) => {
        // return a compiler function
      }
    }
  }
})
```

### Schema Error Formatter

```js
const fastify = Fastify({
  schemaErrorFormatter: (errors, dataVar) => {
    const message = errors.map(e => `${dataVar}${e.instancePath} ${e.message}`).join(', ')
    return new Error(message)
  }
})
```

### Serializer Options

Options passed to `fast-json-stringify`:

```js
const fastify = Fastify({
  serializerOpts: {
    rounding: 'ceil',
    mode: 'standalone'
  }
})
```

### Other Options

| Option | Default | Description |
|---|---|---|
| `useSemicolonDelimiter` | `false` | When `true`, semicolons delimit query string parameters (e.g., `/foo?bar=1;baz=2`). |
| `requireHostHeader` | `true` | When `true`, requests without a Host header receive a 400 error. |

## Server Instance Methods

### .listen(opts)

Starts the server. Returns a promise that resolves to the server address string.

```js
await fastify.listen({ port: 3000, host: '0.0.0.0' })
```

Options:

| Option | Default | Description |
|---|---|---|
| `port` | `0` | Port to bind. `0` assigns a random available port. |
| `host` | `'localhost'` | Address to bind. Use `'0.0.0.0'` for all interfaces. |
| `backlog` | `511` | Max pending connections queue length. |
| `path` | — | Unix domain socket path. Overrides `port` and `host`. |
| `signal` | — | `AbortSignal` to close the server. |
| `listenTextResolver` | — | Custom function to format the "listening" log message. |

```js
// Listen on Unix socket
await fastify.listen({ path: '/tmp/fastify.sock' })

// With AbortSignal
const controller = new AbortController()
await fastify.listen({ port: 3000, signal: controller.signal })
// Later: controller.abort() closes the server

// Custom listen text
await fastify.listen({
  port: 3000,
  listenTextResolver: (address) => `API server running at ${address}`
})
```

### .close()

Gracefully shuts down the server. The shutdown lifecycle is:

1. Set `return503OnClosing` — new requests receive 503
2. Run `preClose` hooks
3. Close the underlying `server`
4. Run `onClose` hooks (cleanup resources)

```js
fastify.addHook('onClose', async (instance) => {
  await instance.db.close()
})
await fastify.close()
```

### .register(plugin, opts)

Register a plugin (encapsulated context). Plugins can add routes, hooks, decorators, and child plugins.

```js
fastify.register(import('./routes/users.js'), { prefix: '/api/users' })

fastify.register(async function (childInstance, opts) {
  childInstance.get('/', async (request, reply) => {
    return { hello: 'world' }
  })
}, { prefix: '/api' })
```

### .after()

Called after the current plugin has finished loading. Useful for adding routes or hooks that depend on a plugin.

```js
fastify
  .register(require('@fastify/postgres'), { connectionString: '...' })
  .after(() => {
    fastify.get('/users', async (request) => {
      const { rows } = await fastify.pg.query('SELECT * FROM users')
      return rows
    })
  })
```

### .ready()

Called when all plugins have loaded. Returns a promise.

```js
await fastify.ready()
console.log('All plugins loaded')
console.log(fastify.printRoutes())
```

### .inject(opts)

Injects a fake HTTP request for testing. Does not require the server to be listening. Returns a promise resolving to a response-like object.

```js
const response = await fastify.inject({
  method: 'GET',
  url: '/api/users',
  headers: { authorization: 'Bearer token' }
})
console.log(response.statusCode) // 200
console.log(response.json())     // parsed JSON body

// POST with payload
const response = await fastify.inject({
  method: 'POST',
  url: '/api/users',
  payload: { name: 'Alice', email: 'alice@example.com' }
})

// With query string
const response = await fastify.inject({
  method: 'GET',
  url: '/api/users',
  query: { page: '1', limit: '10' }
})

// With cookies
const response = await fastify.inject({
  method: 'GET',
  url: '/dashboard',
  cookies: { session: 'abc123' }
})
```

### .addSchema(schema) / .getSchemas() / .getSchema(id)

Add shared JSON schemas that can be referenced by `$ref` in route schemas.

```js
fastify.addSchema({
  $id: 'User',
  type: 'object',
  properties: {
    id: { type: 'integer' },
    name: { type: 'string' },
    email: { type: 'string', format: 'email' }
  },
  required: ['name', 'email']
})

// Reference in route schema
fastify.post('/users', {
  schema: {
    body: { $ref: 'User#' },
    response: { 201: { $ref: 'User#' } }
  }
}, handler)

// Retrieve schemas
const allSchemas = fastify.getSchemas()   // { User: {...} }
const userSchema = fastify.getSchema('User')
```

### .setValidatorCompiler / .setSerializerCompiler

Override the default schema compiler (AJV) or serializer compiler (fast-json-stringify).

```js
// Custom validator using Zod (via fastify-type-provider-zod)
fastify.setValidatorCompiler(({ schema }) => {
  return (data) => {
    const result = schema.safeParse(data)
    if (result.success) return { value: result.data }
    return { error: result.error }
  }
})

// Custom serializer
fastify.setSerializerCompiler(({ schema }) => {
  return (data) => JSON.stringify(data)
})
```

### .setErrorHandler(handler)

Set a custom error handler. Error handlers are encapsulated — a plugin's error handler only handles errors from within that plugin. Unhandled errors propagate to the parent context's error handler.

```js
fastify.setErrorHandler((error, request, reply) => {
  if (error.validation) {
    reply.status(400).send({ errors: error.validation })
    return
  }
  reply.status(500).send({ error: 'Internal Server Error' })
})

// Encapsulated error handlers
fastify.register(async function (instance) {
  instance.setErrorHandler((error, request, reply) => {
    // Handles errors only within this plugin scope
    // Throw or reply.send(error) to propagate to parent
    request.log.error(error)
    reply.status(503).send({ error: 'Service Unavailable' })
  })
})
```

The error handler receives `error`, `request`, and `reply`. The `error` object has these Fastify-specific properties:
- `error.validation` — array of validation errors (when schema validation fails)
- `error.validationContext` — where validation failed: `'body'`, `'params'`, `'querystring'`, `'headers'`
- `error.statusCode` — suggested HTTP status code

### .setNotFoundHandler([opts], handler)

Set a custom handler for 404 responses. Supports per-prefix not-found handlers via encapsulation.

```js
fastify.setNotFoundHandler({
  preHandler: authHook
}, (request, reply) => {
  reply.code(404).send({ error: 'Not Found', message: `Route ${request.url} not found` })
})

// Per-prefix not-found handler
fastify.register(async function (instance) {
  instance.setNotFoundHandler((request, reply) => {
    reply.code(404).send({ error: 'API endpoint not found' })
  })
}, { prefix: '/api' })
```

### .setReplySerializer(serializer)

Set a global serializer for all responses that do not have a response schema defined.

```js
fastify.setReplySerializer((payload, statusCode) => {
  return JSON.stringify({ data: payload, status: statusCode })
})
```

### .setChildLoggerFactory(factory)

Customize how child loggers are created for each request.

```js
fastify.setChildLoggerFactory((logger, bindings, opts, rawReq) => {
  const child = logger.child(bindings, opts)
  child.customMethod = () => {}
  return child
})
```

### .addHook(name, fn)

Add lifecycle hooks. Hooks are encapsulated when registered inside a plugin.

```js
// Request/Reply hooks
fastify.addHook('onRequest', async (request, reply) => {
  request.startTime = Date.now()
})

fastify.addHook('preParsing', async (request, reply, payload) => {
  // Can modify the payload stream
  return payload
})

fastify.addHook('preValidation', async (request, reply) => {
  // Runs before schema validation
})

fastify.addHook('preHandler', async (request, reply) => {
  // Runs after validation, before handler
})

fastify.addHook('preSerialization', async (request, reply, payload) => {
  // Modify response payload before serialization
  return { ...payload, timestamp: Date.now() }
})

fastify.addHook('onSend', async (request, reply, payload) => {
  // Modify serialized payload (string/Buffer)
  return payload
})

fastify.addHook('onResponse', async (request, reply) => {
  const duration = Date.now() - request.startTime
  request.log.info({ duration }, 'request completed')
})

fastify.addHook('onError', async (request, reply, error) => {
  // Notification hook — cannot change the error
  request.log.error(error)
})

fastify.addHook('onTimeout', async (request, reply) => {
  // Fires when requestTimeout is exceeded
})

// Application hooks
fastify.addHook('onReady', async () => {
  // Server is ready but not yet listening
})

fastify.addHook('onListen', async () => {
  // Server is listening
})

fastify.addHook('onClose', async (instance) => {
  // Server is closing — clean up resources
  await instance.db.close()
})

fastify.addHook('preClose', async () => {
  // Before the server starts closing
})

fastify.addHook('onRoute', (routeOptions) => {
  // Called for every registered route — synchronous
  console.log(routeOptions.url, routeOptions.method)
})

fastify.addHook('onRegister', (instance, opts) => {
  // Called for every registered plugin — synchronous
})
```

### .route(opts)

Full route declaration.

```js
fastify.route({
  method: 'POST',
  url: '/api/users',
  schema: {
    body: {
      type: 'object',
      properties: {
        name: { type: 'string' },
        email: { type: 'string', format: 'email' }
      },
      required: ['name', 'email']
    },
    response: {
      201: {
        type: 'object',
        properties: {
          id: { type: 'integer' },
          name: { type: 'string' }
        }
      }
    }
  },
  preHandler: [authHook, rateLimitHook],
  handler: async (request, reply) => {
    const user = await createUser(request.body)
    reply.code(201)
    return user
  }
})
```

### Shorthand Methods

`.get()`, `.post()`, `.put()`, `.delete()`, `.patch()`, `.options()`, `.head()`, `.all()`

```js
fastify.get('/users/:id', {
  schema: {
    params: {
      type: 'object',
      properties: { id: { type: 'integer' } }
    }
  }
}, async (request, reply) => {
  return getUser(request.params.id)
})

fastify.post('/users', async (request, reply) => {
  reply.code(201)
  return createUser(request.body)
})

// .all() matches any HTTP method
fastify.all('/health', async () => ({ status: 'ok' }))
```

### Content Type Parsers

```js
// Custom parser for a content type
fastify.addContentTypeParser('text/xml', { parseAs: 'string' }, (req, body, done) => {
  const parsed = xmlParser.parse(body)
  done(null, parsed)
})

// Async parser
fastify.addContentTypeParser('text/xml', { parseAs: 'string' }, async (req, body) => {
  return xmlParser.parse(body)
})

// Catch-all parser
fastify.addContentTypeParser('*', (req, body, done) => {
  let data = ''
  body.on('data', chunk => { data += chunk })
  body.on('end', () => { done(null, data) })
})

// Check and remove parsers
fastify.hasContentTypeParser('text/xml')          // true/false
fastify.removeContentTypeParser('text/xml')
fastify.removeAllContentTypeParsers()
```

### .getDefaultJsonParser(onProto, onConstructor)

Get the default JSON parser with custom proto/constructor poisoning options.

```js
fastify.addContentTypeParser('application/json-patch+json', {
  parseAs: 'string'
}, fastify.getDefaultJsonParser('remove', 'remove'))
```

### Decorators

Add custom properties to the Fastify instance, Request, or Reply.

```js
// Decorate the server instance
fastify.decorate('db', knex({ client: 'pg', connection: '...' }))
fastify.decorate('authenticate', async (request, reply) => {
  try {
    await request.jwtVerify()
  } catch (err) {
    reply.send(err)
  }
})

// Decorate requests — use getter for per-request values
fastify.decorateRequest('user', null)       // initial value for shape
fastify.decorateRequest('startTime', {
  getter() { return Date.now() }
})

// Decorate replies
fastify.decorateReply('sendSuccess', function (data) {
  this.send({ success: true, data })
})

// Check for decorators
fastify.hasDecorator('db')              // true
fastify.hasRequestDecorator('user')     // true
fastify.hasReplyDecorator('sendSuccess') // true
```

Decorators are encapsulated: decorators added inside a plugin are not visible outside it.

### .log

The Pino logger instance. Available on the server and on each request.

```js
fastify.log.info('Server starting')
fastify.log.error({ err }, 'Something went wrong')

// Per-request logging (includes request ID automatically)
fastify.get('/', async (request) => {
  request.log.info('Handling request')
})
```

### .version

console.log(fastify.version) // '5.8.0'

### .server
